# Narrative Craft Reference

Use this when a draft feels like a report, tutorial, product spec, or explanatory document.

## The Article Test

A publishable personal-experience article should not merely answer "what happened?" or "what is the method?"

It should create a reason to continue reading:

```text
Something was harder / stranger / more useful / more uncomfortable than expected.
```

If the draft can be summarized as "I did X, here are the steps," it needs a stronger narrative arc.

## Opening Test

The opening must create tension before explanation.

Good openings usually start with one of:

- a concrete scene: someone asked, failed, complained, doubted, or saw a result;
- a contradiction: "I thought X, but the real problem was Y";
- a result with boundary: "I got Y result, but not for the reason I expected";
- a mistake: "I started by solving the wrong problem";
- a sharp question: "What do you do when the task is real, urgent, and outside your expertise?"

Avoid openings that sound like:

- "这篇文章想讲..."
- "本文将从..."
- "下面我会分享..."
- "首先我们需要理解..."
- "今天想和大家聊聊..."
- direct method lists before the reader feels the problem.

## Narrative Arc

Before drafting, write one sentence for each beat:

1. **Scene**: the moment that made this experience worth writing.
2. **False belief**: what the author originally misunderstood.
3. **Pressure**: what made the situation difficult or urgent.
4. **Turn**: what changed the author's understanding.
5. **Method**: what the author now does differently.
6. **Boundary**: where the method works and where it does not.
7. **Final judgment**: one sentence the reader can remember.

If a beat is missing, the article will often become a document.

## Section Progression

Each section should do one of these jobs:

- increase tension;
- reveal a new piece of evidence;
- change the reader's understanding;
- make the method usable;
- add a boundary or tradeoff.

Delete or merge sections that merely restate the same thesis.

## Ending Test

The ending should not recap the article.

It should:

- return to the opening tension;
- state one earned judgment;
- avoid repeating the same claim in three different phrasings;
- stop before it becomes motivational or instructional.

Useful ending shapes:

```text
I no longer think X. I now think Y.
```

```text
The point is not that [tool/method] replaces judgment. The point is that it changes when judgment can begin.
```

```text
I used to wait until I understood the domain before starting. Now I use the first version to discover what I need to understand.
```

## Common Failure Modes

- **Explainer drift**: paragraphs explain concepts before the reader cares.
- **Framework inflation**: too many tables, formulas, or checklists too early.
- **Proof dumping**: screenshots are inserted without narrative purpose.
- **Looping conclusion**: the ending repeats "AI is useful but humans judge" several times.
- **Generic uplift**: the final paragraph becomes a motivational slogan.
