## Pattern A: 误判反转型

适合：作者对某件事的理解发生了变化。

```markdown
# 标题：我原本以为 X，后来发现真正重要的是 Y

## 开头：一个反差结论
直接写“我原本以为……后来发现……”

## 背景：为什么我会这样以为
交代经历，不要写太长。

## 现实：事情真正难在哪里
写出冲突、阻力、失败或意外发现。

## 转折：我怎么意识到问题不在 X
给具体事件或判断过程。

## 方法：我现在怎么做
给步骤、清单、模板或判断规则。只有对比关系确实复杂时才用表格。

## 给读者的建议
怎么判断自己是否也遇到这个问题。

## 结尾：更大的判断
把个人经历上升到趋势、能力或方法论。
```

## Pattern B: 失败到方法型

适合：有明显踩坑经历。

```markdown
# 标题：我踩过 XX 的坑后，总结出这套方法

## 开头：失败瞬间
从最具体、最痛的场景开始。

## 我当时错在哪里
列出错误假设，而不是只说“经验不足”。

## 3 个关键坑
每个坑都按：表现 → 原因 → 修正方式。

## 新方法
用流程、短清单或模板呈现；表格只在能明显降低理解成本时使用。

## 使用边界
哪些情况不适合照搬。

## 结尾
失败不是重点，重点是从失败中提炼出判断。
```

## Pattern C: 实践案例型

适合：有具体实践过程、选择、限制、转折和结果的个人经验文章。可以是工具实践、产品设计、运营、教育、职场、创作、生活实验、学习复盘等。

```markdown
# 标题：我做了 XX，真正改变我判断的是 YY

## 项目背景
目标、约束、资源、时间。

## 原始问题
到底要解决什么，不要只写“提升效率”。

## 尝试过程
方案 A/B/C，为什么放弃或保留。

## 最终做法
用步骤、对照表、清单、原则或叙事方式呈现，形式由当前 source 决定。

## 关键判断
哪些地方不是表面变量，而是更深的限制、判断、关系或能力问题。

## 可复制模板
如果适合，给读者一个可复用的模板；如果不适合模板化，就给判断边界。

## 结尾
从案例看能力变化。
```

## Pattern D: 小白解释型

适合：把复杂经验讲给非专业读者。

```markdown
# 标题：如果你也刚开始做 XX，先搞懂这几个问题

## 开头：小白最容易误解的点
让读者马上觉得“这就是我”。

## 简化模型
用 1 张图或 1 个比喻解释。

## 例子
用作者自己的经历说明。

## 操作步骤
3-5 步即可。

## 常见坑
用问答/清单呈现。

## 结尾
鼓励行动，但不要空喊口号。
```

## Pattern E: 趋势洞察型

适合：个人经验能反映一个行业、职业、关系、学习方式、创作方式或生活方式的变化。

```markdown
# 标题：从我做 XX 这件事，我看到一个变化

## 开头：个人经历中的异常信号
写一个具体瞬间。

## 这个信号为什么重要
连接行业或普通人的变化。

## 它改变了什么
工作方式、关系结构、能力结构、判断方式、机会门槛或生活选择。

## 我的实践
用自己的案例证明不是空谈。

## 未来建议
读者接下来可以训练什么能力。

## 结尾
给出谨慎但有前瞻性的判断。
```
